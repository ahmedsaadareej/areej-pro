## ⚡ قواعد التنفيذ

**قبل البدء:**
- لا تلمس أي ملف قبل الموافقة
- لا تعدّل ملفات خارج النطاق المعلن إلا بإذن صريح
- لا تبدأ مهمة جديدة قبل إغلاق الحالية بـ commit

**أثناء التنفيذ:**
- عدّل الملفات المعلنة فقط
- لو اكتشفت إن مهمة تانية مرتبطة — أخبرني أولاً، لا تنفذها
- اكتب كود واضح مع comments عربية على أي منطق غير بديهي

**بعد التعديل:**
- Syntax check أولاً: `node --check <file>` لكل ملف JS جديد
- Smoke check: `pm2 reload areej-pro && sleep 3 && curl -s http://localhost:3002/health`
- لو نجح: commit واحد بالرسالة: `feat(inbox-v4): [وصف موجز]`
- لو فشل: إصلاح مرة واحدة فقط — لو استمر الفشل: `git checkout -- <files>` ثم أخبرني

```

- الحالة: [مكتملة / توقف بسبب ...]
- ما تم: [ملفات + وصف موجز]
```
